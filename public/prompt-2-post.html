Prompt 2 Post: AWS Serverless with Bedrock, Lex, and Rekognition

Chat with a bot, the system finds your photo, generates captions with Claude, and posts it to Bluesky.

If you want to dig into the code, the project is up on GitHub (in my teammate Swetha's repo). The full IEEE write-up is also available as a PDF.

Prompt 2 Post is a cloud-native pipeline that automates the creation and posting of captioned social media images. A user signs in, types something like "post a beach photo with a chill caption," and the system finds a matching image from their connected Google Photos library, generates a caption using a foundation model, overlays the caption on the image, and publishes it to Bluesky. The whole loop runs on AWS with no servers to manage and no human in the middle once the prompt is sent.

Team project, four people. The captioning brain runs on Amazon Bedrock with Claude 3 Haiku, the image understanding on Rekognition, and the matching layer on OpenSearch.

System architecture

Two pipelines do the heavy lifting. The ingestion pipeline runs daily on EventBridge: a Coordinator Lambda fans out one SQS job per user, a Photo Fetcher Lambda pulls fresh images from Google Photos using OAuth refresh tokens stored in Secrets Manager, drops them into S3, and an Image Indexing Lambda runs Rekognition for labels and pushes metadata into OpenSearch. The posting pipeline starts at the chatbot: a Lex V2 bot on a static S3 frontend captures imageTopic and captionStyle slots, a backend Lambda uses Claude to clean the input and queries OpenSearch for a matching image, then queues the job to SQS. Downstream Lambdas call Claude for short and long captions, render the overlay with Pillow, and a final Lambda authenticates to Bluesky (credentials in Secrets Manager, JWT cached in ElastiCache) and posts the image.

Caption generation went through real iteration. The team evaluated open-source models like BLIP and Flan-T5, then several Bedrock options, before settling on Claude 3 Haiku for its multimodal input and clean structured-JSON output.

Stack: AWS Lambda for every compute step, Amazon Bedrock with Claude 3 Haiku for caption generation, Amazon Lex V2 for conversational input, Cognito and Secrets Manager for auth and credentials, S3 for image storage and the static frontend, Amazon Rekognition for image labeling, OpenSearch for indexing image metadata, SQS (with Dead Letter Queues) for queueing between stages, EventBridge for scheduled triggers, ElastiCache for token caching, and API Gateway for the OAuth and chat endpoints.
