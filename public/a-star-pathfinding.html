A* Pathfinding

A* finds the shortest path between two points on a grid by combining the cost of getting to a node with an estimate of the cost remaining. The version below adds 8-directional movement and an angle-change penalty, so the algorithm trades raw shortness for smoother, less zig-zaggy paths when you turn the penalty up.

How it works

  1. Open set, ordered by f. A* keeps a priority queue of candidate nodes, sorted by f = g + h, where g is the cost from the start and h is a heuristic estimate of cost to the goal. The lowest-f node is always expanded next.
  2. Octile heuristic. For 8-directional movement, the heuristic uses max(dx, dy) + (sqrt(2) - 1) * min(dx, dy). It is admissible and tight, which keeps the search focused.
  3. Step cost with diagonal weighting. Cardinal moves cost 1, diagonals cost sqrt(2). Diagonal squeezes through two adjacent walls are blocked so paths cannot phase through corners.
  4. Angle-change penalty. When you arrive at a node from direction A and leave in direction B, the algorithm adds (angle_between(A, B) / pi) * penalty to the step cost. With the slider at 0 you get a standard A* path. Crank it up and the path straightens out, sometimes accepting a longer route to avoid sharp turns.

Try setting the penalty to 0, drawing a few obstacles, and running. Then bump the penalty to 2 and run again on the same map. The frontier expansion looks similar but the final path is visibly straighter.
