8-Puzzle Solver

I built an A* solver for the classic sliding-tile puzzle in Python with two admissible heuristics: pure Manhattan distance (h1) and a tighter Manhattan plus linear conflict variant (h2). The interactive demo below ports the solver to JavaScript so you can pick a starting position, choose a heuristic, and watch the optimal solution play out as tiles slide into place. Toggling between h1 and h2 on the same scramble shows how a sharper heuristic explores far fewer nodes for the same answer.

How A* solves it

  1. States and successors. The current arrangement of tiles is a state. Successors come from sliding the blank one square left, right, up, or down. Each move has cost 1.
  2. h1 (Manhattan distance). Sum of the row and column distances each tile is from its goal position. Admissible because every tile must travel at least that far.
  3. h2 (linear conflict). Adds 2 for every pair of tiles in the same goal row or column that are in reverse order. Two such tiles cannot pass each other without one stepping aside, so the extra moves are unavoidable. The estimate stays admissible and gets noticeably tighter.
  4. Best-first search by f = g + h. A min-heap orders frontier states by their f value. A closed set prevents re-expanding states that have already been visited, turning the search into A* graph search.

Try the preset puzzle with h1 first, then run it again with h2. The solution depth is identical, but the node count drops noticeably because h2 prunes branches that h1 still thinks are worth exploring.
