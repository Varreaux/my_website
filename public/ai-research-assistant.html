AI Research Assistant

Have you tried OpenAI's research mode? This is my take on recreating that functionality. You give it a topic, a pipeline of specialized agents takes over, and a few minutes later a structured research report lands in your inbox.

How it works

  1. Planner agent. Breaks your query into 3 targeted web searches, each with a stated reason. Uses Pydantic models for structured output.
  2. Search agent. Fans those searches out in parallel via asyncio, calling OpenAI's WebSearchTool and condensing each result into a 2 to 3 paragraph summary under 300 words.
  3. Writer agent. Takes the summaries plus the original query, drafts an outline, and produces a full markdown report alongside a short summary and follow-up research questions.
  4. Email agent. Converts the markdown into clean HTML and ships it via SendGrid using a @function_tool decorator.

Under the hood

A ResearchManager orchestrates the whole pipeline. Async generators stream progress updates to the UI as each stage completes, and OpenAI's tracing is wired in so you can watch exactly what each agent is doing in real time. The front end is Gradio, so the whole thing is one query box and a live status feed.

It is a compact case study in coordinating specialized agents, running parallel async work, and passing structured data cleanly between AI systems.

Try it

Live demo: huggingface.co/spaces/Varreaux/deep_research

Heads up: it takes a couple minutes to run, and while you will see the output on screen, I also get an email with your search results.
