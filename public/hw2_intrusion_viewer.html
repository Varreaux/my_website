Probe Attack Detection

I built a logistic regression classifier to detect probe-style network attacks in the KDD Cup '99 intrusion dataset. My full code is in the notebook further down the page. The hard part is class imbalance: probe attacks make up just 0.83% of the half-million records, so plain accuracy looks great no matter what the model does, and the real story is hiding in the recall on the minority class. The notebook walks through training the baseline classifier, comparing L1 versus L2 regularization, identifying the most informative network features, and then revisiting everything after rebalancing the classes through undersampling. The headline lesson is that undersampling fixes recall but trades it for a precision crash, which is itself a useful negative result.

Try it

Below is a 2D version of the same dynamics on a synthetic dataset. KDD '99 is 41-dimensional, but the imbalance behavior reproduces cleanly in 2D, where you can actually see what the model is doing. Drag the imbalance slider down toward 5% and watch accuracy stay near 99% while recall on the minority class collapses. Then flip Undersample to balance and watch recall come roaring back while precision crashes.
