Kropki Sudoku Solver

I built a constraint satisfaction solver in C++ for a variant of Sudoku where dots between cells encode extra rules: a white dot means the two values differ by 1, a black dot means one is double the other. The standard Sudoku rules still apply on top. The solver uses backtracking with the MRV and degree heuristics, plus forward checking against both the Sudoku and Kropki constraints. The interactive demo below is a JavaScript port that animates the search so you can watch the algorithm pick variables, try values, and back out of dead ends.

How the solver picks moves

  1. MRV (minimum remaining values). Pick the empty cell with the smallest legal domain. Fewer choices means less branching, and contradictions surface earlier.
  2. Degree heuristic tiebreaker. When multiple cells tie on MRV, pick the one whose value would constrain the most other empty cells (most empty neighbors in its row, column, and 3x3 block).
  3. Forward consistency check. Each candidate value is tested against current Sudoku rules and the Kropki dot constraints before being placed, so the search never enters a state that already violates a constraint.
  4. Backtrack on failure. If no value works, undo the last assignment and try the next candidate one level up. The animation flashes the cell red on the way out.

Try the Pure Kropki preset. The grid starts completely empty, so every value has to be derived from the dot constraints alone. The solver leans heavily on MRV here because the dots create cells with very small initial domains.
