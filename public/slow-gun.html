Slow Gun

Dual-Weapon Action Platformer

Click HERE to play the game on itch. It's browser-based, so very easy to play.

A 2D action platformer built in Godot that explores tactical combat through a weapon-switching system and a slow-down mechanic. The goal was to design encounters where players had to slow things down in order to progress.

The core mechanic is a dual-weapon system: players can switch between a gun and a sword, each with different combat applications. The gun fires projectiles for ranged combat, deals no damage but you already know what it does ;) The sword creates area-of-effect attacks. The tactical element comes from the slow-down mechanic, when player bullets hit turrets, they reduce the turret's firing rate by dynamically adjusting timer intervals, creating strategic choices about target priority. Player bullets also slow down enemies.

I implemented the weapon switching using an enum-based system that tracks available weapons and current selection. The slow-down mechanic required timer management: stopping the timer, adjusting wait times, and restarting it without breaking the firing cycle. This taught me about state management in real-time systems and how small mechanical interactions can create meaningful gameplay depth.

The game also features enemy AI with horizontal pursuit behavior, where enemies track the player's position and move toward them. Breakable doors that respond to sword attacks (but not bullets) added another layer of interaction, requiring collision detection between area-based attacks and static objects.

The checkpoint and respawn system was straightforward but important for player experience. It stores spawn positions and resets player state.
