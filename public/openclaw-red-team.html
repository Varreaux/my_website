Red-Teaming AI Agents on OpenClaw

An adversarial dashboard that probes a live agent for prompt-injection weaknesses.

If you want to dig into the code, the whole project is up on my GitHub.

Live AI agents that touch sensitive data are a fast-growing attack surface. If an agent is wired up to confidential records, can it be coaxed into leaking them, abandoning its instructions, or following a stranger's commands instead of the operator's? This project is a red-team harness for putting that question to the test against an agent running on Lightning AI's OpenClaw gateway. The demo scenario is a fictional housing-assistance organization whose AI assistant has access to client mortgage records, but the same harness would work against any OpenClaw-hosted agent.

The dashboard fires categorized attacks at the agent over a single click: social engineering and role-override claims, chain-of-thought elicitation, tool-use manipulation, censorship-rewrite requests, and prompt injections embedded in content that looks like user data. Each attack runs in its own session so the agent has no memory of previous tries. Responses are scored against a refusal-keyword list and labeled BLOCKED, BYPASSED, or ERROR, and a live attack flow animates each hit in real time so you can watch the agent hold or fold round by round.

The backend is FastAPI talking to OpenClaw over WebSocket with Ed25519 device authentication, and the frontend is vanilla HTML, CSS, and JavaScript with no framework dependencies. The keyword-based scorer has a known limitation worth flagging: it catches refusals expressed in conventional language but mislabels assertive or reframing refusals as bypasses, where the agent says something like "I don't have safety filters to disable" rather than "I can't disable safety filters." Swapping the keyword scorer for an LLM judge that evaluates semantic compliance is the natural next iteration.
